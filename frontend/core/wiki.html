1. 项目概述

竞赛特攻队 - 竞赛考试必胜系统是一个专为编程和AI竞赛学习者设计的在线学习平台，支持C++、Python、Go、Scratch等多种编程语言，包含学习计划、消课系统、线上编程等功能模块。系统支持三种用户角色：学生、教师和管理员，每种角色拥有不同的权限。

2. 系统架构

2.1 前端

前端使用HTML、CSS和JavaScript构建，包含以下主要文件：

  • auth/login.html - 登录和注册页面，支持用户登录和注册功能，包含角色选择（学生/教师）
  • core/study_plan.html - 学习计划表页面，展示8周冲刺学习计划，包含学习周期表、代码模板、笔记功能等
  • core/wiki.html - 项目Wiki页面，提供系统功能介绍和使用指南
  • course/course_system.html - 消课系统页面，包含学员管理、上课记录、消课记录功能
  • programming/online_programming.html - 线上编程页面，提供C++代码编译和执行环境
  • user/profile.html - 个人空间页面，支持用户资料编辑和头像上传
  • materials/real_test_question_materials/ - 存放历年真题材料，包含2023年9月至2025年9月的真题PDF文件
  • tests/test_features.html - 功能测试页面，用于测试系统功能
  • tests/test_all_features.html - 全功能测试页面，用于测试系统的所有功能
  • tests/code_examples.html - 代码示例页面，提供C++代码示例

2.2 后端

后端使用Flask框架构建，包含以下主要文件：

  • app.py - 应用主文件，配置应用、初始化扩展、注册路由
  • extensions.py - 扩展初始化文件，初始化数据库、JWT和CORS
  • decorators.py - 装饰器文件，包含JWT认证装饰器
  • config/.env - 环境配置文件，包含数据库连接信息和其他配置
  • requirements.txt - 依赖包文件，列出项目所需的Python包
  • start_server.py - 服务器启动脚本
  • setup_mysql.py - MySQL数据库设置脚本
  • test_flask.py - Flask测试脚本
  • test_minimal.py - 最小化测试脚本
  • test_api.py - API测试脚本
  • test_compile.py - 编译功能测试脚本
  • test_output.py - 输出功能测试脚本
  • models/ - 数据模型目录
    • __init__.py - 模型包初始化文件
    • models.py - 数据模型文件，定义用户、笔记、学生、课程记录等模型
  • routes/ - 路由目录，包含认证、笔记、用户管理、编译和课程系统路由
    • __init__.py - 路由包初始化文件
    • auth.py - 认证路由，处理用户登录和注册
    • notes.py - 笔记路由，处理笔记的增删改查
    • users.py - 用户路由，处理用户管理
    • compile.py - 编译路由，处理C++代码编译和执行
    • course.py - 课程系统路由，处理学员管理、上课记录和消课记录
  • scripts/ - 脚本目录
    • create_test_accounts.py - 创建测试账号脚本
    • init_db.py - 数据库初始化脚本
  • instance/gesp_study_plan.db - SQLite数据库文件

2.3 数据库

使用SQLite数据库（默认），包含以下主要表：

  • users - 用户表，存储用户信息和权限
    • id - 用户ID（主键）
    • username - 用户名（唯一）
    • password - 密码（加密存储）
    • email - 邮箱（可选，唯一）
    • nickname - 昵称（可选）
    • role - 角色（student/teacher）
    • admin - 是否为管理员（布尔值）
    • created_at - 创建时间
    • updated_at - 更新时间
  • notes - 笔记表，存储用户笔记
    • id - 笔记ID（主键）
    • user_id - 用户ID（外键，关联User表）
    • title - 笔记标题
    • content - 笔记内容
    • type - 笔记类型（text/image/mindmap）
    • images - 图片路径数组（JSON格式）
    • created_at - 创建时间
    • updated_at - 更新时间
  • students - 学生表，存储学生信息
    • id - 学生ID（主键）
    • teacher_id - 教师ID（外键，关联User表）
    • name - 学生姓名
    • gender - 性别
    • age - 年龄
    • grade - 年级
    • project - 项目
    • created_at - 创建时间
    • updated_at - 更新时间
  • class_records - 课程记录表，存储上课记录
    • id - 记录ID（主键）
    • student_id - 学生ID（外键，关联Student表）
    • teacher_id - 教师ID（外键，关联User表）
    • date - 日期
    • start_time - 开始时间
    • end_time - 结束时间
    • topic - 主题
    • content - 内容
    • created_at - 创建时间
    • updated_at - 更新时间
  • course_records - 消课记录表，存储消课记录
    • id - 记录ID（主键）
    • class_id - 课程记录ID（外键，关联ClassRecord表）
    • student_id - 学生ID（外键，关联Student表）
    • status - 状态（completed/cancelled/missed）
    • notes - 备注
    • created_at - 创建时间
    • updated_at - 更新时间

数据库配置

  • SQLite配置（默认）：数据库文件 `instance/gesp_study_plan.db`，连接字符串 `sqlite:///gesp_study_plan.db`
  • MySQL配置（可选）：连接字符串 `mysql+pymysql://root:password@localhost/gesp_study_plan`，需在.env文件中修改DATABASE_URL配置

3. 核心功能

3.1 学习计划表

学习计划表是系统的核心功能，提供8周冲刺学习计划，包含：

  • 学习周期表：展示每周的学习主题和日期
  • 核心知识点：详细列出每周需要掌握的知识点
  • 真题实战：提供历年真题和重点题目
  • 常用代码模板：提供编程常用的代码模板
  • 笔记功能：支持添加、查看和搜索笔记
  • 根据用户角色显示或隐藏消课系统按钮

3.2 消课系统

消课系统面向教师和管理员，包含：

  • 学员管理：添加、删除学员信息
  • 上课记录：添加上课记录
  • 消课记录：查看所有消课记录
  • 管理员设置：只有管理员可以访问，用于管理教师账户权限

3.3 线上编程

线上编程功能提供真实的C++代码编译和执行环境，支持：

  • 代码编写：在线编写C++代码，支持语法高亮
  • 代码编译：使用g++编译器真实编译C++代码
  • 代码执行：运行编译后的程序并查看结果
  • 用户输入：支持程序运行时的用户输入
  • 错误处理：显示编译错误和运行时错误
  • 代码示例：提供C++代码示例供学习参考

3.4 认证功能

认证功能负责用户身份验证和权限管理，包含：

  • 用户注册：支持创建新用户，可指定角色（学生/教师）
  • 用户登录：验证用户身份，生成JWT令牌
  • 权限控制：基于角色和管理员权限的访问控制

3.5 笔记功能

笔记功能支持用户记录和管理学习笔记，包含：

  • 创建笔记：支持文本、图片和思维导图类型的笔记
  • 查看笔记：获取用户的所有笔记
  • 更新笔记：修改现有笔记
  • 删除笔记：删除笔记及其关联的图片
  • 图片上传：支持上传图片作为笔记附件

3.6 用户管理

用户管理功能仅对管理员开放，包含：

  • 用户列表：获取所有用户信息
  • 用户详情：获取单个用户信息
  • 更新用户：修改用户信息和权限
  • 删除用户：删除用户账号

3.7 个人空间

个人空间功能提供用户资料管理，包含：

  • 资料编辑：修改用户昵称、邮箱等信息
  • 头像上传：上传和更新用户头像
  • 个人信息展示：显示用户基本信息

4. 权限控制

4.1 学生端

  • 可以访问学习计划表
  • 可以使用线上编程功能
  • 看不到消课系统

4.2 教师端

  • 可以访问学习计划表
  • 可以使用线上编程功能
  • 可以使用消课系统
  • 看不到管理员设置

4.3 管理端

  • 可以访问所有功能
  • 可以管理用户账号和权限
  • 可以访问管理员设置

5. 使用指南

5.1 登录系统

  1. 打开 auth/login.html 页面
  2. 输入用户名和密码
  3. 点击登录按钮

5.2 注册账号

  1. 打开 auth/login.html 页面
  2. 点击"立即注册"链接
  3. 填写用户名、密码和邮箱
  4. 选择用户角色（学生/教师）
  5. 点击注册按钮

5.3 测试账号

系统已预创建以下测试账号，可直接使用：

用户名 密码 角色 权限
demo 123456 管理员 访问所有功能
student student123 学生 访问学习计划、线上编程
teacher teacher123 教师 访问学习计划、线上编程、消课系统

5.4 启动服务器

  1. 进入 backend 目录
  2. 运行 python app.py 命令
  3. 服务器默认运行在 http://127.0.0.1:5000

6. 技术栈

6.1 前端

  • HTML5
  • CSS3
  • JavaScript

6.2 后端

  • Python 3.14
  • Flask
  • SQLAlchemy
  • JWT
  • CORS
  • bcrypt（密码加密）
  • python-dotenv（环境变量管理）

6.3 数据库

  • SQLite（默认）
  • MySQL（可选）

6.4 服务器启动

  1. 确保安装了所有依赖包：pip install -r requirements.txt
  2. 运行服务器：python app.py
  3. 服务器默认运行在 http://127.0.0.1:5000

6.5 数据库初始化

  1. 系统启动时会自动创建数据库表结构
  2. 会自动创建demo管理员账号（用户名：demo，密码：123456）

6.6 数据安全

  • 密码使用bcrypt加密存储
  • JWT令牌用于身份验证
  • 基于角色和管理员权限的访问控制
